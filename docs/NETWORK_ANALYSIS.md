# Network Analysis

The routing network is modeled as pgRouting edges in `roads`.

- `source` and `target` are network node ids generated by pgRouting topology.
- `cost` and `reverse_cost` remain the legacy distance cost in meters.
- `walk_cost_s`, `bike_cost_s`, and `drive_cost_s` are travel-time costs in seconds.
- Reverse cost columns model directionality. For example `oneway=yes` sets `reverse_drive_cost_s = -1`.

## Modes

- `walk` uses walking-speed costs and can use pedestrian links such as `footway` and `path`.
- `bike` uses bicycle-speed costs and blocks roads tagged as unavailable to bicycles.
- `drive` uses inferred or explicit road speeds and respects one-way direction.

## Service Area Polygon

`/api/analyze` calls `pgr_drivingDistance`, joins reached nodes back to road edges, buffers the reachable roads, unions them, simplifies the geometry, and returns both:

- `serviceArea`: buffered polygon.
- `reachableRoads`: road features used to create the polygon.

Default buffer width is mode-dependent: walk 40 m, bike 65 m, drive 125 m.
